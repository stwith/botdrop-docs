:::caution[实验性功能]
自动化是一个**探索性功能**，整个过程中可能会遇到各种问题：
- 执行不稳定，有时会失败
- Bot 可能会"犯傻"，点错位置或误解指令
- 不同 App 的兼容性参差不齐
- **自动化操作可能触发 App 的风控机制，存在封号风险**

**建议：**
- 使用**闲置的安卓手机**进行自动化，避免影响主力机
- 使用**小号**测试，降低封号带来的损失
- 使用更智能的模型（如 Claude Sonnet/Opus、Codex-5.3）可以提高成功率
- 把常用的操作流程沉淀为自定义 Skills，让 Bot 按固定步骤执行，而不是每次都靠 AI 临场发挥
:::

完成[自动化配置](/zh/automation/)后，你的 AI Bot 可以使用 Python 和 uiautomator2 控制手机 UI。本页介绍如何使用这个技能。

## 能做什么

自动化技能让你的 Bot 可以：
- 打开和操作各种 App
- 点击按钮和菜单项
- 输入文字（支持中文）
- 读取屏幕内容
- 执行复杂的多步骤工作流

## 前置条件

使用自动化命令前，确保：

1. **Shizuku 正在运行**
2. **u2 服务正在运行**

## 基本用法

:::tip[首次使用]
Bot 需要先了解自己的能力才能执行自动化操作。首次使用时，告诉它："读一下 botdrop-automation 这个 skill"。Bot 会读取技能文档并理解如何使用。
:::

直接告诉 Bot 你想在手机上做什么。例如：

- "打开微信，给张三发一条消息：晚上一起吃饭"
- "截个屏给我看看"
- "打开设置，看看电池还剩多少"
- "打开小红书，发一张图片，配文'今日份快乐'"

Bot 会把复杂任务拆解成一步步的操作，每一步都会观察屏幕状态。

## 工作原理

自动化遵循 **观察 → 操作 → 观察** 的模式：

1. **观察**：读取当前屏幕状态（哪个 App、显示了什么）
2. **操作**：执行一个动作（点击、输入、滑动）
3. **观察**：检查结果，再决定下一步

这种逐步执行的方式确保了可靠性，特别是对于动态变化的 App 界面。

## 支持的操作

### 导航
- 按名称打开 App
- 点击按钮、菜单项、图标
- 滑动和滚动
- 按返回/主页键

### 文字输入
- 在搜索框、聊天输入框、表单中输入
- 通过 FastInputIME 支持中文输入
- 清除和替换文字

### 读取屏幕
- 获取当前 App 和页面
- 检查元素是否存在
- 读取屏幕上的文字内容

## 示例

### 打开 App
```
"打开微博"
```
Bot 会找到并启动微博 App。

### 发送消息
```
"打开微信，找到妈妈的聊天，发一条：晚上回家吃饭"
```
Bot 会：
1. 打开微信
2. 搜索妈妈的聊天
3. 点击进入对话
4. 输入消息
5. 点击发送

:::note
微信自动化目前还在探索阶段，效果可能不稳定，欢迎反馈使用体验。
:::

### 查看信息
```
"打开设置，告诉我手机的安卓版本"
```
Bot 会导航到关于手机页面并读取版本信息。

## 使用技巧

### 描述要具体
与其说"在社交媒体发点东西"，不如说"打开小红书，发一张最近的照片，配文'周末愉快'"。

### 给足时间
复杂的操作需要时间。Bot 会等待页面加载并验证每一步。

### 容错处理
如果出了问题，Bot 会尝试恢复或询问你的指示。

## 限制

- **不输入密码**：出于安全考虑，不要让 Bot 输入密码
- **App 差异**：有些 App 有特殊的自定义 UI，可能需要特别处理
- **需要网络**：App 操作需要网络连接
- **屏幕要亮**：自动化需要屏幕处于活动状态

## 常见问题

### "u2 service not running"
去自动化面板点击 "Start u2 Service"。

### 操作似乎没有生效
- 确保目标 App 完全加载
- 尝试更具体地描述要点击什么
- 检查 App 是否需要更新

### 文字输入失败
- Bot 会自动尝试不同的输入方式
- 中文输入使用 FastInputIME 以保证可靠性

### Bot 好像卡住了
- 说"停止"或"取消"来中断
- 检查 App 是否弹出了对话框
